Guide to Skeleton 2.0.4

Skeleton is a simple and lightweight CSS style sheet. It provides a set of classes that can be used to define the overall grid structure (rows and columns) in which to put content on web pages. The "Using Skeleton" section of this document explains how to use the classes in the Skeleton grid system. The "Customizing Skeleton" section discusses the classes in detail and reviews how the "custom" version, in use here, was customized. Skeleton can be downloaded for free at getskeleton.com.

Using Skeleton

There are three main structural classes in Skeleton: .container, .row, and .columns. These classes are nested in such a way as to form a system of rows and columns in which to place content. It is best to create structure using these classes in neutral div tags, and to put the actual content in semantic tags nested inside of the structural classes. However you can use these classes in any block-level tag if it makes more sense to do it that way in a particular situation.

The .container class is meant to be applied to the outer div and forms the outer container of the content. It holds the rows and columns that form the grid structure. Within the .container div there are one or more .row divs that form the rows. Each row is divided into 12 equal-width columns that can be used in any combination using the twelve .column classes:

  • .one.columns
  • .two.columns
  • .three.columns
  • .four.columns
  • .five.columns
  • .six.columns
  • .seven.columns
  • .eight.columns
  • .nine.columns
  • .ten.columns
  • .eleven.columns
  • .twelve.columns

Below is an example of how to set up the grid structure. In this example there is one outer container and one row containing three 4-column width columns.

<div class="container">
   <div class="row">
      <div class="four columns">
         <p>Four columns"</p>
      </div>
      <div class="four columns">
         <p>Four columns"</p>
      </div>
      <div class="four columns">
         <p>Four columns"</p>
      </div>
   </div> <!-- close row div -->
</div>    <!-- close container div -->

In addition to the grid, Skeleton comes with a few handy utility classes.

  • .u-full-width {width: 100%; box-sizing: border-box;}
  • .u-max-full-width {max-width: 100%; box-sizing: border-box;}
  • .u-pull-right {float: right;}
  • .u-pull-left {float: left;}
  • .u-cf {content: ""; display: table; clear: both; }

The image in the left column is using class="u-full-width". This class scales the image to always take up 100% the width of its container.
The image in the left column is using class="u-max-full-width". This class will allow the image to scale down if it's container is smaller than the original image size, but it won't scale up larger than the original width of the image.
The image to the right is using class="u-pull-right", causing the image to float right. It is not recommended to use the u-max-full-width class in conjunction with u-pull-left or u-pull-right.
The image to the left is using class="u-pull-left", causing the image to float left.

Notice that "max-width" creates a fluid image. The "width" attribute defines the width of specific element. The "max-width" attribute defines the maximum size the element is allow to have.

For example, width:100% uses the parent's width to calculate current width value whereas max-width:100% use its own original width to calculate the maximum size. So, the image with width: 100% could be larger its original size (scaled base on its parent width). On the other hand, the image with max-width: 100% could be smaller but never be scaled larger its original size (maximum valid width = 100% x original width). That's why it's called fluid image.

Skeleton is Mobile-first, and comes with media queries set up for five break points.

  • Larger than mobile -- @media (min-width: 400px) {}
  • Larger than phablet -- @media (min-width: 550px) {}
  • Larger than tablet -- @media (min-width: 750px) {}
  • Larger than desktop -- @media (min-width: 1000px) {}
  • Larger than Desktop HD -- @media (min-width: 1200px) {}

Customizing Skeleton

Skeleton is meant to be a starting point that you can customize to your needs. In this section we look at the particular customization of Skeleton in use here.

  1. Original Skeleton assigns box-sizing: border-box to .container and .column separately. We combine them in to one rule and add .row. We also added the vendor prefixes for the box-sizing property as follows: 
        .container, .row, .columns {
      box-sizing: border-box;
      /* add vendor prefixes for box-sizing */
      -moz-box-sizing: border-box;
      -webkit-box-sizing: border-box;
    }
  2. We changed the overall size of the original Skeleton .container from 960px max-width with 20 pixels of left and right padding and 100% width, to 1000px max-width with zero padding and 98% width.
  3. We changed the left-margin on .columns from 4% to .8%
  4. The creators of Skeleton created a number of .column classes (singular of .columns) to make the singular columns, like .one and .one-third, grammatically correct. We eliminated all of the singular .column forms as we found them to unnecessarily complex and potentially confusing.
  5. We added a rule to make .row 100% width: .row {width:100%;}
  6. We removed all of the base styles included in Skeleton for HTML, H1-H6, BODY, P, A, UL, OL, LI, CODE, PRE, HR and the TABLE tags since all of these are handled in the reset and will be styled separately for our website in the main.css file.
  7. We replaced the custom Button and Form styles in Skeleton with our own Form and Button styles.
  8. Skeleton column widths are percentage-based to make them fluid. Since we changed the overall width and padding of the .container class as well the width of the gaps between columns, we needed to recalculate the widths on all of the .columns classes. We show the formula for calculating column widths below, as well as how to create custom column widths combinations outside of the 12-column grid.

Overall width is based on number of columns (COLS), column width (COL-WIDTH), and gutter width (GUTTER-WIDTH). The formula is as follows: 

    WIDTH = (COLS * COL-WIDTH) + ((COLS - 1)) * GUTTER-WIDTH)

In our case we have 12 columns, each column is 76 pixels wide, and the space between columns (gutter) is 8 pixels. Our formula looks like this: 

    1,000 pixels = (12 columns * 76 pixels) + (11 columns * 8 pixels)

In order to be responsive, we convert pixels to percentages in our class rules. Each column is: (76 pixels / 1,000 pixels) = 7.6% of the total width. Likewise, the gutters are: (8 pixels / 1,000 pixels) = .8% of the total width. We used this to set the left margin on the .columns class to .8%.

To calculate the column width in percentages for each of the column-width classes, use the following calculation: 

  ((76 * COLUMNS) + (8 * (COLUMNS - 1)) /1,000) / 100

If you want to create a different set of columns, beyond the 12 equal-width columns in the grid systems, you can use these formulas for any combination of pixel widths you need.

For example, let's say we want a two column section where the left column is 225 pixels with our 8 pixel gap in between it and the right column which takes the remaining 767 pxels. We will name our columns .left-225 and .right-767. Here are the styles and percentages we will create to produce these columns. Note that these columns can only be used with each other in any given row. 

  .left-225.columns {width: 22.5%;} /* (225/1000) x 100 */
  .right-767.columns {width: 76.7%} /* (767/1000) x 100 */
